## 六、项目经验（改进版）

```
基于知识图谱的四川方言智能问答系统

项目简介：
从零构建包含 26,468 个节点和 39,742 条关系的领域知识图谱，
覆盖方言词库、场景对话、意图映射、文化实体四个子图，
实现"文本输入→意图识别→知识图谱检索→大模型生成→方言纠错"的8层流水线架构。

技术栈：
FastAPI + Neo4j + DeepSeek大模型 + React + Git

核心成果：

1. 知识图谱构建
   - 四子图分域架构：方言词汇5,165个、用户语料10,836条、意图映射530个、文化实体2,125个
   - 关系网络：21种关系类型，累计39,742条边，支持跨子图语义关联
   - 数据规模：26,468节点、39,742关系，数据完整率98%

2. 系统性能
   - API平均延迟：30ms（不含LLM）
   - 知识图谱查询延迟：10-50ms
   - 端到端响应时间：平均1.8秒（含DeepSeek API调用）
   - 支持6大类意图识别：美食/旅游/天气/交通/翻译/闲聊

3. 技术实现
   - 8层处理流水线：输入标准化→意图识别→场景匹配→实体抽取→知识检索→模板匹配→响应生成→方言纠错
   - 方言纠错模块：自动修正"撒子"→"啥子"等常见错误
   - 前后端分离：FastAPI异步后端 + React动态前端
```
